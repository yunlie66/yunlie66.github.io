PELRO（Relocation Read Only）重定向只读
​在Linux系统安全领域数据可以写的存储区就会是攻击的目标，尤其是存储函数指针的区域。 所以在安全防护的角度来说尽量减少可写的存储区域对安全会有极大的好处.

GCC，GNU linker以及Glibc-dynamic linker一起配合实现了一种叫做RELRO的技术。大概实现就是由linker指定binary的一块经过dynamic linker处理过relocation之后的区域为只读。用来防御hijack GOT攻击。

设置符号重定向表格为只读或在程序启动时就解析并绑定所有动态符号，从而减少对GOT（Global Offset Table）攻击。

在Linux中有两种RELRO模式：Partial RELRO 和 Full RELRO。Linux中Partical RELRO默认开启。

  • Partial RELRO

    • 该ELF文件的各个部分被重新排序。内数据段（internal data sections）（如.got，.dtors等）置于程序数据段（program’s data sections）（如.data和.bss）之前；
    • 无 plt 指向的GOT是只读的；
    • GOT表可写

  • Full RELRO

    • 支持Partial模式的所有功能；
    • 整个GOT表映射为只读的。

函数return了一个 read(0, buf, 0x100uLL);也就是回了一个read操作，大小为0x100uLL也就是100个空间，那是不是也就意味着只要我们把这100个空间打满，然后追加上对应的内容，溢出的部分就能被程序read到。

双击buf

我们可以得到三个地址：

两者相加，一共0x28个地址，把这些个地址用字符堵死，后续就能伪造执行的read

把0x28个空间填满溢出然后把backdoor函数地址用p64(0x4011FB)拼接上，就可以让read拿到我们的bin/sh权限了。

一开始的思路是makemoney直到赚够9999$购买shell，但”All things are only available for one day!”，只能赚一天24hour的钱，肯定赚不够9999$。

后来注意到int v1和unsigned __int64 v2

16位系统中一个int能存储的数据的范围为-3276832767，而unsigned int能存储的数据范围则是065535，在计算机中，整数是以补码形式存放的。

1、int 是整数类型，用于定义变量的类型，有符号unsigned int 是无符号的整数类型，直白点说有符号无符号整型就是能不能存放负数。

2、根据程序编译器的不同，整形定义的字节数不同。注意一下取值范围。

C语言中：unsigned int a=-2;printf(“%u”,a);输出结果为多少？

输出结果为65534或者4294967294

因为在不同位的操作系统下，数值的表示位数是不一样的。首先我们先明确一点：在计算机中，正数用原码来计算(正数的原码、补码、反码都是一样的)；负数是用补码来计算的。至于原因可以参考一下文章网页链接就是把减法简化为加法从而大大简化计算机电路的过程。

首先在16位系统中，-2的原码为1000 0000 0000 0010；补码为：1111 1111 1111 1110；我们使用的unsigned为无符号数，那么此时计算机会把我们的-2强制转化为正数进行运算，即用1111 1111 1111 1110表示-2，即为65534.

在32位操作系统中，-2的原码为：1000 0000 0000 0000 0000 0000 0000 0010；补码为：1111 1111 1111 1111 1111 1111 1111 1110；同上述，计算机会强制输出1111 1111 1111 1111 1111 1111 1111 1110所代表的十进制数，即4294967294。

于是尝试使money为负数换算为unsigned int型可以购买shell

原文链接：mmap函数的用法详解及实例分析-CSDN博客

mmap函数是一个比较神奇的函数，它可以把文件映射到进程的虚拟内存空间。通过对这段内存的读取和修改，可以实现对文件的读取和修改，而不需要用read和write函数。如下图所示，为mmap实现原理的示意图。

在这个图中，off为映射的部分在文件中的偏移量，len为映射的长度。

下面我们来看一下mmap函数的原型

参数addr：指定映射的起始地址，通常设为NULL，由内核来分配

参数length：代表将文件中映射到内存的部分的长度。

参数prot：映射区域的保护方式。可以为以下几种方式的组合：

PROT_EXEC 映射区域可被执行
PROT_READ 映射区域可被读取
PROT_WRITE 映射区域可被写入
PROT_NONE 映射区域不能存取

参数flags：映射区的特性标志位，常用的两个选项是：

MAP_SHARD：写入映射区的数据会复制回文件，且运行其他映射文件的进程共享

MAP_PRIVATE：对映射区的写入操作会产生一个映射区的复制，对此区域的修改不会写会原文件

参数fd：要映射到内存中的文件描述符，有open函数打开文件时返回的值。

参数offset：文件映射的偏移量，通常设置为0，代表从文件最前方开始对应，offset必须是分页大小的整数倍。

函数返回值：实际分配的内存的起始地址。

参考链接：第一周pwn的题目_buf = mmap((void *)0x233000, 0x1000ull, 7, 34, -1,-CSDN博客

buf = (void *)(int)mmap((void *)0x66660000, 0x1000uLL, 7, 50, -1, 0LL);这里是将buf的数据位置写入mmap分配的位置0x66660000

原文链接：PWN入门（1-1-5）-pwntools学习 (yuque.com)

context是pwntools用来设置环境的功能。在很多时候，由于二进制文件的情况不同，我们可能需要进行一些环境设置才能够正常运行exp，比如有一些需要进行汇编，但是32的汇编和64的汇编不同，如果不设置context会导致一些问题。

一般来说我们设置context只需要简单的一句话:

这句话的意思是：

  1. os设置系统为linux系统，在完成ctf题目的时候，大多数pwn题目的系统都是linux
  2. arch设置架构为amd64，可以简单的认为设置为64位的模式，对应的32位模式是’i386’
  3. log_level设置日志输出的等级为debug，这句话在调试的时候一般会设置，这样pwntools会将完整的io过程都打印下来，使得调试更加方便，可以避免在完成CTF题目时出现一些和IO相关的错误。

查看main函数伪代码

  1. seed = time(0LL)：使用time函数获取当前时间的秒数，并将其赋值给seed变量。
  2. srand(seed)：使用seed作为随机数的种子，初始化随机数生成器。
  3. v8 = rand()：生成一个随机数，并将其赋值给v8变量。
  4. __isoc99_scanf("%d", &v6)：从标准输入读取一个整数，并将其存储到v6变量中。
  5. qmemcpy(v9, "2$031", sizeof(v9))：将字符串”2$031”的内容复制到v9数组中。
  6. v3 = (char)v9[rand() % 5]：将v9数组中一个随机位置的字符转换为char类型，并将其赋值给v3变量。
  7. sy((unsigned int)(char)v9[v4 % 2], v3)：调用一个名为sy的函数，将(char)v9[v4 % 2]和v3作为参数传递给它。

再查看sy函数伪代码

这段代码是一个名为sy的函数，返回类型为unsigned __int64，表示函数的执行结果。下面是对代码的解释：

  1. char command[2]; // [rsp+16h] [rbp-Ah] BYREF：长度为2的字符数组command，通过引用传递的方式在栈上分配内存。
  2. unsigned __int64 v4; // [rsp+18h] [rbp-8h]：无符号64位整数类型的变量v4，用于存储一个值。
  3. v4 = __readfsqword(0x28u)：将当前线程的GS寄存器中的值读取到v4变量中。
  4. command[0] = a1：将参数a1的值赋给command数组的第一个元素。
  5. command[1] = a2：将参数a2的值赋给command数组的第二个元素。
  6. system(command)：调用system函数，参数为command数组，执行command中存储的命令。
  7. return v4 - __readfsqword(0x28u)：计算v4减去当前线程的GS寄存器中的值，并将结果作为返回值返回。

缓冲区溢出：在main函数中，通过qmemcpy函数将字符串”2$031”复制到长度为5的缓冲区v9中。然后在调用sy函数时，将v9数组中的某个字符作为参数传递给sy函数。由于v9数组有5个字节的空间，而sy函数中的command数组只有2个字节的空间，因此存在缓冲区溢出漏洞。攻击者可以通过精心构造的输入，将恶意的代码注入到command数组中，从而执行任意命令。
